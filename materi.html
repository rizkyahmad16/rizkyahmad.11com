Penggabungan String
Baris 16–17 digunakan document.writeln untuk menampilkan pesan selamat datang baru. Ekspresi di dalam tanda kurung menggunakan operator+ untuk "menambahkan" string (harfiah "Halo, "), variabel nama (string yang dimasukkan pengguna di baris 14) dan string lain (literal ", selamat datang di pemrograman JavaScript!"). JavaScript memiliki versi+ operator untuk rangkaian string yang memungkinkan string dan nilai tipe data lain (termasuk string lain) untuk digabungkan. Hasil dari operasi ini adalah string baru (dan biasanya lebih panjang). Jika kita berasumsi bahwanama berisi string literal "Jim", ekspresi mengevaluasi sebagai berikut: JavaScript menentukan bahwa dua operan yang pertama + operator (string "Halo," dan nilai variabel nama) keduanya adalah string, lalu menggabungkan keduanya menjadi satu string. Selanjutnya, JavaScript menentukan bahwa dua operan dari operator + kedua (hasil dari operasi penggabungan pertama, string" Halo, Jim", dan rangkaian ", selamat datang di pemrograman JavaScript!") keduanya adalah string dan menggabungkan keduanya. Ini menghasilkan string" Halo, Jim, selamat datang di program JavaScriptming!". Browser merender string ini sebagai bagian dari dokumen HTML5. Perhatikan bahwa ruang antaraHalo, dan Jim adalah bagian dari string "Halo, ".

   Seperti yang akan Anda lihat nanti, operator + yang digunakan untuk penggabungan string dapat mengonversi tipe variabel lain menjadi string jika perlu. Karena penggabungan string terjadi di antara dua string, JavaScript harus mengonversi tipe variabel lain menjadi string sebelum dapat melanjutkan operasi. Prioritas Operator
JavaScript menerapkan operator dalam ekspresi aritmatika dalam urutan yang tepat ditentukan oleh berikut: aturan prioritas operator, yang umumnya sama dengan yang diikuti dalam aljabar:
1. Operasi perkalian, pembagian dan sisa diterapkan pertama. Jika suatu ekspresi berisi beberapa operasi perkalian, pembagian, dan sisa, operator diterapkan dari     kiri ke kanan. Operasi perkalian, pembagian, dan sisa dikatakan memilikitingkat prioritas yang sama.
2. Operasi penjumlahan dan pengurangan diterapkan selanjutnya. Jika ekspresi berisi beberapa operasi penambahan dan pengurangan, operator diterapkan darikiri     ke kanan. Operasi penjumlahan dan pengurangan memilikitingkat prioritas yang sama.

Aturan prioritas operator memungkinkan JavaScript untuk menerapkan operator dalam urutan yang benar. Persamaan yang Ketat (===) dan Ketat Tidak Sama dengan (!==)Operator
Seperti yang kami sebutkan di Bagian 6.5, JavaScript dapat mengonversi antar tipe untuk Anda. Ini termasuk kasus di mana Anda membandingkan nilai. Misalnya perbandingan"75" == 75 menghasilkan nilai benar karena JavaScript mengonversi string "75" ke nomor 75 sebelum melakukan pemerataan (==) perbandingan. Untuk mencegah konversi implisit dalam perbandingan, yang dapat menyebabkan hasil yang tidak diharapkan, JavaScript menyediakan persamaan yang ketat (===) dan ketat tidak sama (!==) operator. Perbandingan"75" === 75 menghasilkan nilai Salah karena satu operan adalah string dan yang lainnya adalah angka. Demikian pula,75" !== 75 hasil benar karena jenis operan tidak sama, oleh karena itu nilainya tidak sama. Jika Anda tidak menggunakan operator ini saat membandingkan nilai denganbatal, 0, benar, Salah atau string kosong (""), javascriptlint.comValidator JavaScript menampilkan peringatan potensi konversi implisit.

• Terkadang berguna untuk menampilkan informasi di jendela yang disebut dialog (atau kotak dialog) yang "muncul" di layar untuk menarik perhatian pengguna.    Dialog biasanya menampilkan pesan penting kepada pengguna yang menjelajahi halaman web.
• Peramban jendela objek menggunakan metode peringatan untuk menampilkan dialog peringatan.
• Urutan pelarian \n adalah karakter baris baru (hal. 192). Ini menyebabkan kursor dalam dokumen HTML5 bergerak ke awal baris berikutnya.

Bagian 4 Memperoleh Masukan Pengguna dengan mengingatkan Dialog
• Kata kunci adalah kata-kata dengan arti khusus dalam JavaScript.
• Kata kunci var di awal pernyataan menunjukkan bahwa kata nama adalah variabel. Variabel adalah lokasi di memori komputer di mana nilai dapat disimpan untuk    digunakan oleh skrip. Semua variabel memiliki nama dan nilai, dan harus dideklarasikan dengan pernyataan var sebelum digunakan dalam skrip.
• Nama variabel dapat berupa pengenal valid apa pun yang terdiri dari huruf, angka, garis bawah ( _ ) dan tanda dolar ($) yang tidak dimulai dengan angka dan    bukan merupakan kata kunci JavaScript yang dicadangkan.
• Deklarasi diakhiri dengan titik koma dan dapat dibagi menjadi beberapa baris dengan setiap variabel dalam deklarasi dipisahkan dengan koma—dikenal sebagai    daftar nama variabel yang dipisahkan koma. Beberapa variabel dapat dideklarasikan dalam satu deklarasi atau dalam beberapa deklarasi.
• Sangat membantu untuk menunjukkan tujuan variabel dalam skrip dengan menempatkan komentar JavaScript di akhir deklarasi variabel. Komentar satu baris    dimulai dengan karakter // dan berakhir di akhir baris. Komentar tidak menyebabkan browser melakukan tindakan apa pun saat skrip ditafsirkan; sebaliknya,    komentar diabaikan oleh juru bahasa JavaScript.
• Komentar multibaris dimulai dengan pembatas /* dan diakhiri dengan pembatas */. Semua teks di antara pembatas komentar diabaikan oleh penerjemah.
• NS jendela milik objek mengingatkan metode menampilkan dialog di mana pengguna dapat mengetikkan nilai. Argumen pertama adalah pesan (disebut prompt)    yang mengarahkan pengguna untuk mengambil tindakan tertentu. Argumen opsional kedua, dipisahkan dari yang pertama dengan koma, dapat menentukan    string default yang akan ditampilkan di bidang teks.
• Variabel diberi nilai dengan penugasan, menggunakan operator penugasan, =. NS = operator disebut operator biner, karena memiliki dua operan (hal. 196).
• JavaScript memiliki versi + operator untuk penggabungan string yang memungkinkan string dan nilai tipe data lain (termasuk string lain) untuk digabungkan.

Bagian 5 Konsep Memori
• Setiap variabel memiliki nama, tipe dan nilai.
• Saat sebuah nilai ditempatkan di lokasi memori, nilai tersebut menggantikan nilai sebelumnya di lokasi tersebut. Ketika sebuah nilai dibacakan dari lokasi memori,    prosesnya tidak merusak.
• JavaScript tidak memerlukan variabel untuk memiliki tipe yang dideklarasikan sebelum dapat digunakan dalam skrip. Variabel dalam JavaScript dapat berisi nilai    dari tipe data apa pun, dan dalam banyak situasi, JavaScript secara otomatis mengonversi antara nilai dari tipe yang berbeda untuk Anda. Untuk alasan ini,    JavaScript disebut sebagai bahasa yang diketik secara longgar.
• Ketika sebuah variabel dideklarasikan dalam JavaScript, tetapi tidak diberi nilai, variabel tersebut memiliki nilai yang tidak ditentukan. Mencoba menggunakan nilai    variabel seperti itu biasanya merupakan kesalahan logika.
• Ketika variabel dideklarasikan, mereka tidak diberi nilai default, kecuali jika Anda menentukannya. Untuk menunjukkan bahwa variabel tidak mengandung nilai,    Anda dapat menetapkan nilaibatal untuk itu.

Bagian 6 Aritmatika
• Operator aritmatika dasar (+, -, *, /, dan %; P. 200) adalah operator biner, karena masing-masing beroperasi pada dua operan.
• Tanda kurung dapat digunakan untuk mengelompokkan ekspresi dengan cara yang sama seperti dalam ekspresi aljabar.
• JavaScript menerapkan operator dalam ekspresi aritmatika dalam urutan tepat yang ditentukan oleh aturan prioritas operator berikut.
• Ketika kita mengatakan bahwa operator diterapkan dari kiri ke kanan, kita mengacu pada asosiatifitas operator. Beberapa operator mengasosiasikan dari kanan ke    kiri.

Bagian 7 Pengambilan Keputusan: Operator Kesetaraan dan Relasional
• JavaScript jika pernyataan memungkinkan skrip untuk membuat keputusan berdasarkan benar atau salahnya suatu kondisi. Jika kondisi terpenuhi, pernyataan    dalam tubuh jika pernyataan dijalankan. Jika kondisi tidak terpenuhi (yaitu, kondisi salah), pernyataan di tubuhjika pernyataan tidak dieksekusi
• Kondisi di jika pernyataan dapat dibentuk dengan menggunakan operator kesetaraan dan operator relasional.
